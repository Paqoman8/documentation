## ğŸ“– About

This project serves as a **Gold Standard Reference** for structuring a GitHub repository. It includes all the necessary documentation files, templates, and configuration to ensure a project is maintainable, accessible, and community-friendly.

While the code itself is a simple Vanilla JS portfolio, the primary value lies in the **documentation architecture**.

## âœ¨ Features

- **Complete Documentation Suite**: Includes `CONTRIBUTING`, `CODE_OF_CONDUCT`, `SECURITY`, and more.
- **GitHub Templates**: Pre-configured Issue and Pull Request templates.
- **Standardized Changelog**: Follows "Keep a Changelog" principles.
- **Clean Architecture**: Separation of concerns in a vanilla environment.

## ğŸš€ Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) (Optional, only for running a local dev server if desired)
- A modern web browser

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/Paqoman8/documentation.git
   ```

2. **Navigate to the project directory**
   ```bash
   cd documentation
   ```

3. **Open `index.html`**
   Simply open the `src/index.html` file in your browser to view the project.

## ğŸ’¡ Usage

This template is meant to be forked or used as a reference when setting up new repositories.

1. Copy the `docs/` folder and root documentation files to your new project.
2. Customize the content (replace "Vanilla JS Portfolio" with your project name).
3. Enjoy a professional-looking repo from Day 1.

For more detailed instructions, see [USAGE.md](docs/USAGE.md).
