## Reporting a Vulnerability

We take the security of our software very seriously. If you find a vulnerability, please report it to us immediately.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them by sending an email to [security@example.com]. You should receive a response within 24 hours. If for some reason you do not, please follow up by email to ensure we received your original message.

We prefer all communications to be encrypted using PGP. Key ID: `XXXXXXXX`

### Process

1. **Report**: You report the vulnerability privately.
2. **Triage**: We confirm the vulnerability and determine its severity.
3. **Fix**: We develop a fix and test it.
4. **Release**: We release a security update.
5. **Disclosure**: We publicly disclose the vulnerability after a reasonable time has passed to allow users to upgrade.
